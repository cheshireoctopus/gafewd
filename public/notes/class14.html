Review: Function Arguments

• So far, named functions have been defined like this:


	function alertName () {
	  alert('Keanu Reeves')
	}

• What happens if we want to alert another name?

• We would have to declare another function:


	function alertAnotherName () {
	  alert('Gary Busey');
	}

• Is this scalable? Say we wanted to alert the entire cast of Point Break (1991)...


	function alertBodhi () {
	  alert('Patrick Swayze');
	}



	function alertJohnny () {
	  alert('Keanu Reeves');
	}



	function alertPappas () {
	  alert('Gary Busey');
	}



	function alertTyler () {
	  alert('Lori Petty');
	}

Only 37 more functions to write...

Review: Function Arguments

• Instead, we could write:


	function alertName (nameInput) {
	  alert(nameInput);
	}

	alertName('Keanu Reeves');
	alertName('Patrick Swayze');

• Function arguments make our code:

⇒ Cleaner

⇒ Reusable

⇒ More maintainable

Review: Multiple Arguments

• Functions can take more than one argument (up to 255!)

• Define multiple arguments using a comma-separated list:


	function add (x, y, z) {
	  var sum = x + y + z;
	  alert(sum);
	}

• And called like so:


	add(1,2,3);
	// alerts 6

	add(5,10,15);
	// alerts 30

<form> Element

• Forms are used to capture user input

• User input is sent to (processed by) the server

• HTML forms can contain a variety of elements:

⇒ inputs and textareas

⇒ selects (dropdowns)

⇒ checkboxes

⇒ radio buttons

⇒ submit buttons

<form> Element

• Example form structure:


	<form>
		<input type="text" placeholder="Name">

		<select>
			<option value="one">One</option>
			<option value="one">Two</option>
		</select>

		<input type="checkbox"> Checkbox 1

	</form>

.submit() Event

.submit() Event

• Fires when <form> elements are submitted


	$('#my-form').submit(handleFormSubmit);

	function handleFormSubmit () {
	  // do something!
	}

• On submit, will send form data to server (this requires configuration)

• Only works on <form> elements

• More info on <form> attributes

.submit() Event

• Specifically, .submit() will fire when:

⇒ User clicks <input type="submit>

⇒ User clicks <button type="submit></button>

⇒ User presses enter key while a <form> element is selected

Preventing Default Event Behaviors

Preventing Default Event Behavior

• Some events/elements have default behaviors

• For example, when clicked on, the following link will take you to http://google.com


	<a href="http://google.com">Google</a>

• Often, developers will want to override these default behaviors

Preventing Default Event Behavior

• Override defaults using .preventDefault()


	$('#my-form').submit(handleFormSubmit);

	function handleFormSubmit (event) {
	  event.preventDefault();

	  // your code here!
	}

• The event argument can be passed into the function associated with the event

• You may also see this referred to as: evt, or even: e
