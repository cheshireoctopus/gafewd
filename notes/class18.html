Flexbox

Flexbox: Overview

• As you are probably aware...HTML/CSS page layout can be difficult

• Flexbox, a new layout mode in CSS3, is a tool for making 2D layouts (e.g. column vs. row) easier

• Main idea: give a container the ability to alter (flex) its childrens' sizing to best-fill the available space

• Bonus: containers can easily vertically/horizontally align their children

Flexbox: Overview

• Two components: a flex container and its flex children

Flexbox: Overview

• The equivalent in HTML:


	<div class="container">
		<div class="blob"></div>
		<div class="blob"></div>
		<div class="blob"></div>
	</div>

• The <div class="container"> represents the flex container

• The <div class="blob">'s represent the flex children

Flexbox: Overview

• To use flexbox, apply display: flex; as a CSS rule to the flex container

• Once applied, all of the flex container's immediate children become flex children

• The flex children should now be rendering inline (as if they were floating)

• Codepen example

Flexbox: Containers - Horizontal Alignment

• Use the justify-content CSS property to horiztonally align a flex container's children:

⇒ flex-start (default): items align at the left

⇒ flex-end: items align at the right

⇒ center: items are centered

⇒ space-between: items are evenly distributed with items aligning at the conatiner's start/end

⇒ space-around: items are evenly distributed in the line with equal space around them

Flexbox: Containers - Vertical Alignment

• Use the align-items CSS property to vertically align a flex container's children:

⇒ flex-start: items aligned at the top of the container

⇒ flex-end: itemts aligned at the bottom of the container

⇒ center: items are centered

⇒ baseline: items are aligned along the baseline of their respective text

⇒ stretch (default): items stretch to fill their container

CSS Frameworks

CSS Frameworks: Overview

• Standardized approach for solving common problems

• Pre-packaged code used to support web development

• Goals:

⇒ Provide a common structure (boilerplate)

⇒ Eliminate repetitive work

⇒ Speed development process

⇒ Increase on-boarding time of new developers

CSS Frameworks: Overview

Frontend Framework:

• Contains standardized code and assets (HTML/CSS/JS)

• Typography and pre-built styles for HTML elements

• CSS grid for developing a responsive layout

• Solutions to cross-browser compatibility issues

• JS library for UI/UX interactions (dropdowns/carousels/etc.)

CSS Frameworks: Overview

Two types of framework:

1) Grid/Layouts (simple):

⇒ 960 Grid System

⇒ Toast

⇒ Base

2) Complete (complex):

⇒ Bootstrap

⇒ (Zurb) Foundation

⇒ Skeleton

CSS Frameworks: Selecting a Framework

Rule of thumb:

⇒ The more stylistically complex (the more custom CSS you will be writing), the less complex of a framework you will need

As with plugins:

⇒ Open source

⇒ Recently updated

⇒ Community of active contributors

⇒ Well-documented

CSS Frameworks: Pros/Cons

Pros:

⇒ Major increase in speed of development

⇒ Solutions to common problems already built

⇒ Cross-browser compatibility

Cons:

⇒ Reliance: you might un-learn CSS!

⇒ Dependence: what if you want to switch frameworks?

⇒ Bloat: unused framework code

⇒ Identical/similar-looking/clones

Bootstrap

Bootstrap: What is it?

In their own words:

"Bootstrap is the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web."
Bootstrap: What is it?

• Best thing to come out of Twitter

• OG built by @mdo and @fat

• CSS framework (with HTML and JS components)

• Collection of CSS styles for common UI components:

⇒ Navbars

⇒ Buttons

⇒ Forms

⇒ Image Carousels

• Notable for its responsive grid layout

Bootstrap: What is it?
Bootstrap: The Grid

Source: Girl Develop It

Bootstrap: The Grid

Core CSS Classes:

.container

⇒ Top-level element; contains other HTML

.row

⇒ Horiztonal row used to hold .col's

⇒ Must be children of a .container

.col

⇒ Used to hold content (text/images/etc.)

⇒ Must be immediate children of row's

Bootstrap: Columns

• Bootstrap's grid is divided into 12 columns

• Columns are created using CSS classes

• Build columns using the following formula: col-{size}-{width}

• <div class="col-md-6">

⇒ A <div> for medium-sized screens that is 6-columns wide

• <div class="col-xs-2">

⇒ A <div> for x-small screens that is 2-columns wide

Bootstrap Components: Overview

• Make it easy to implement typical functionality:

Bootstrap Components: Navbar

• Bootstrap's Navbar provides a responsive nav out-of-the-box

• Provides built-in links, search, and dropdown components

• Can be "fixed" to top/bottom of screen

• Easy-install:

1) Copy the starter code from the documents

2) Paste the starter code right before the opening <body> tag

3) Customize to your liking

Bootstrap Components: Navbar
Bootstrap Components: Navbar

• The nav's containers:


	<nav class="navbar navbar-inverse navbar-fixed-top">
		<div class="container-fluid">
			...rest of nav elements here
		</div>
	</nav>

• Apply the following classes to nav:

⇒ navbar

⇒ navbar-inverse

⇒ navbar-fixed-top

• Creates a black navbar that is fixed to the top of the page

Bootstrap Components: Navbar

• The nav's primary children (nested inside container-fluid):

			<span class="sr-only">Toggle navigation</span>
			<span class="icon-bar"></span>
			<span class="icon-bar"></span>
			<span class="icon-bar"></span>
		</button>
		<a class="navbar-brand" href="index.html">
			<img src="img/your-logo-here.png" width="20" heigh="20" alt="your logo">
