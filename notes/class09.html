JS Reivew

document.querySelector()

⇒ Queries HTML for the first element with the specified CSS selector


	document.querySelector('#logo')

⇒ The above will look through a HTML document and return the first element with id="logo"

⇒ Once an element is returned, additional JavaScript methods can be "called" on it

⇒ Can use variables to store refrences to HTML elements:


	var logo = document.querySelector('#logo');

	logo.onclick = someFunction;
JS Reivew

• Querying Examples:


	// will return the first element with an ID of "logo"
	document.querySelector('#logo');

	// will return the first <h1> element
	document.querySelector('h1');

	// will return all <p> elements
	document.querySelectorAll('p');

	// will return all elements with a class of "feature"
	document.querySelectorAll('.feature');

• Note: document.querySelectorAll returns a list (an array) of DOM nodes

• Will be covering the list/array data structure next week

• Functions

⇒ Functions are chunks of code, grouped together, that execute (or run) line-by-line

⇒ Think of them as miniature programs


	function coolPersonAlert () {
	   var coolPerson = 'Maria';
	   alert(coolPerson + 'is a cool person.');
	}

1. What is the name of this function?

2. What does this function do?

3. Does this function have any state (anything kept in memory)?

4. How would you call, or invoke, this function?

Functions: Named vs. Anonymous

• Named Functions:

⇒ Function is associated with a name

⇒ Useful if you plan on using the function in many places


	function calculateTax () {
	  // func body
	}

• Anonymous Functions:

⇒ Function is created and executed on the fly

⇒ Useful for callbacks and event handlers


	document.querySelector('#my-el').onclick = function () {
	  // func body
	}
Functions (ctn)

• Functions can be associated with variables...


	var doSomething = function () {
	  alert('do something!');
	}

• ...and then used to respond to browser events:


	document.querySelector('#some-btn').onclick = doSomething;

• NOTE: If stored in a variable, define functions before calling them:


	// WOULD NOT WORK!
	document.querySelector('#some-btn').onclick = doSomething;

	var doSomething = function () {
	  alert('do something!');
	}

Browser Events

Apply events in three steps:

1) Use document.querySelector() to select the element you want to apply the event to:


	document.querySelector('#click-me')

2) Apply the event name (in this ex, .onclick) to the selected element:


	document.querySelector('#click-me').onclick

3) Set the above equal to the function that will run when the event is triggered


	document.querySelector('#click-me').onclick = function () {
	  // do something
	}
Browser Events: Click Event

• Use .onclick to add a click handler to an element

• Events can be handled with both anonymous and named functions:

• Named function:


	document.querySelector('#some-el').onclick = doSomething;

	function doSomething () {
	  // function body
	}

• Anonymous function:


	document.querySelector('#some-el').onclick = function () {
	  // function body
	}

Browser Events: Mouse Over/Out

• Use .onmouseover and .onmouseout to achieve a hover effect:


	var button = document.querySelector('#my-btn');

	button.onmouseover = overFunction;
	button.onmouseout = outFunction;

⇒ .onmouseover is triggered when a user moves the cursor over the desired HTML element

⇒ .onmouseout is triggered when a user's cursor leaves said element

.innerHTML Property

.innerHTML Property

• .innerHTML is used to read and write HTML

• Assuming the following HTML:


	<div id="result">Hello, World!</div>

⇒ To read:


	document.querySelector('#result').innerHTML;
	=> 'Hello, World!'

⇒ To write:


	document.querySelector('#result').innerHTML = 'Cats';

parseInt() Function

parseInt() Function

• .innerHTML, and similar methods that pull text from HTML, return these values as strings!

• parseInt() converts the string data type to a number data type


	var x = "2"

	x + 2
	=> "22"

	x = parseInt(x)
	x + 2
	=> 4

<select> element

.value Property

.value Property

• .value is used to get/set <input> values

⇒ Read:


	document.querySelector('#my-input').value

⇒ Write:


	document.querySelector('#my-input').value = 'some value'

• WATCHOUT - similar to .innerHTML

.value Property

• .value can also be used on <select> elements

• Uses the <option>'s value attribute


	<select id="my-dropdown">
	  <option value="option-one">Beer</option>
	  <option value="option-two">Coffee</option>
	  <option value="option-three">Water</option>
	</select>

• The following will return the above's selected <option> value!


	document.querySelector('#my-dropdown').value

• Returns "option-one", NOT "Beer"
