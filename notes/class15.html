Responsive Web Design - Review

RWD:

Designing the optimal viewing experience no matter the device on which the site is being viewed

Static vs. Fluid Layouts:

Using %-based widths, margins, etc. instead of px-based

Media Queries:

Allow CSS rules targeted at specific variables, like screen-size

• When width > 1000px color text red

• When width < 1000px color text blue

Browser-Width Media Queries

Within CSS Stylesheet:


	@media screen and (min-width: 1024px) {
		color: red;
	}

Media Type: screen

Only target devices with a viewable screen

Rule: (mid-width: 1024px)

Only target windows larger than, or equal to, 1024px

Media Queries - Additional Examples

Don't have to specify a media type:


	@media (min-width: 700px) { ... }

Multiple rules:


	@media (min-width: 320px) and (max-width: 480px) { ... }

• These styles will only apply on devices between 320 and 480 pixels

Em Typography

EM ... What?

• An em is a unit of measurement

• Derived from print design

• Based on the size of the letter "M"

• Like pixels, they determine the size of elements on a web page

• Unlike pixels, which are fixed, ems are relative to their parent's font-size

• More Info

EM's

• Repeat: em's are relative to their parent's font-size

• 1em === the inherited font size

• If a <div> has font-size of 16px:

1em within said <div> is 16px

2em within said <div> is 32px

• If a <p> has font-size of 20px:

1em within said <p> is 20px

2em within said <p> is 40px

EM's

Example:


	body {
		font-size: 20px;
	}

	div {
		height: 5em; // 100px
		width: 3em; // 60px
		margin: 0.5em; // 10px
		border: 0.05em; // 1px
	}

EM's - General Guide Lines

1. Set a font-size on the <body> using pixels:


	body {
		font-size: 16px;
	}

2. Base other typography on ems:


	h1 {
		font-size: 3em; // 48px
	}

	h2 {
		font-size: 2em; // 32px
	}

3. Set font-size in media queries to be smaller:


	h1 {
		font-size: 1.5em; // 32px;
	}

<meta> Tag

<meta> Tag

• Mobile browsers render webpages in a window wider than the screen, the viewport

• This is why you often have to pinch, zoom, and pan to move around a mobile browser

• To ensure that your site is actually being viewed at the width of the device, and not the viewport, use:


	<meta name="viewport" content="width=device-width, initial-scale=1">

<meta> Tag


	<meta name="viewport" content="width=device-width, initial-scale=1">

"width=device-width"

Controls the size of the viewport; sets it to the device's width

"initial-scale=1"

Controls the zoom level

Responsive Rules of Thumb

Responsive Rules of Thumb

• Use pixels for font size on <body> and other top-level elements (containers, wrappers, etc.)

• Use percents for widths, margins, padding of of major layout elements

• Use em's for typography: <h1>, <p>, etc.

• Include the viewport <meta> in the <head>
